Maximum Security — Personal Cyber Hygiene

Simple rules. Strict habits. Fewer breaches.

Passwords & Vault+
  • Strength: ≥15 chars; upper/lowercase, numbers, special characters.
  • Content: no personal info; no readable words/phrases.
  • Uniqueness: different password per platform/account.
  • Rotation: change every 1–3 months; never reuse last 5 (prefer never reuse).
  • Storage: never in browser autofill, notes, screenshots, or paper.
  • Sharing: never share; avoid shoulder-surfing.
  • Manager: KeePass 2 (offline DB) with strong master password; add keyfile/2FA if available.
MFA & Passkeys+
  • Prefer passkeys; otherwise app-based TOTP (no SMS).
  • Store recovery codes offline; register two authenticators minimum.
Sessions & Devices+
  • Auto-logout after 5–10 minutes idle where possible.
  • Lock screen when away (Win+L / Ctrl+Cmd+Q).
  • Disable Bluetooth/Wi-Fi when not needed on phones and PCs.
Email, Links & Attachments+
  • Do not open unsolicited photos, GIFs, PDFs, Office files, ZIPs, installers.
  • If the sender has weak awareness, double-check before opening anything.
  • Scan suspicious URLs with VirusTotal.
  • Analyze headers: MXToolbox.
  • Check exposures: Have I Been Pwned.
Networks & Connectivity+
  • Avoid public Wi-Fi; if unavoidable, use a trusted VPN and avoid sensitive accounts.
  • Do not use personal accounts on networks you don’t control.
  • Avoid free/unknown services; read security and privacy terms.
Data Handling & Privacy+
  • Prefer E2E encryption and pre-shared keys for sensitive data.
  • Remove metadata (author, GPS, device) before sharing files.
  • Verify integrity via SHA-256 hashes when possible.
Sandbox & Monitoring+
  • Use a sandbox or VM to test untrusted files/links first.
  • Watch for CPU/network spikes, new services, or autostarts.
Social Media Exposure+
  • Avoid posting routines, locations, or photos that aid profiling or security-answer guessing.
System Update Policy & Platform Guidance+
  • Windows: stay on latest stable release (e.g., Windows 11 25H2 when available). Secure Boot and TPM enabled.
  • Ubuntu/Linux: stay on the latest LTS or current stable release supported by your apps.
  • Android/iPhone: keep OS and apps updated; remove unused apps.
  • Programs: keep software on the latest version.
  • Update SLA: Normal/Mid: ≤7 days; High: ≤48 hours; Security/Critical: ≤24 hours.
Device Hardening+
  • Full-disk encryption (BitLocker/FileVault/mobile encryption).
  • UEFI Secure Boot on; BIOS/UEFI admin password; TPM enabled.
  • Separate standard vs admin accounts; least privilege daily use.
Mobile Security+
  • Disable sideloading; verify publishers; revoke unused permissions.
  • Long PIN/passcode; Find My Device; auto-wipe after N failures.
Browser & Extensions+
  • Block third-party cookies; clear site data on close for risky profiles.
  • Only necessary, reputable extensions; HTTPS-only; block WebRTC leaks if needed.
Home/Small Office Network+
  • Router: change default admin; disable WPS/UPnP; enable WPA3; separate guest SSID.
  • Update router firmware; consider DNS filtering.
Backups (3-2-1)+
  • 3 copies, 2 media, 1 offsite; ransomware-resistant snapshots.
  • Encrypt backups; test restores quarterly.
Phishing Playbook+
  • Beware MFA fatigue, QR-phish, homoglyph domains, urgent/CEO lures.
  • Flow: Stop → Validate → Report.

Quick Commands

Windows: Get-FileHash -Algorithm SHA256 <file>
Linux: sha256sum <file>
macOS: shasum -a 256 <file>
Strip metadata: exiftool -all= <file>

Quick Routines

  • Daily: lock devices; avoid unexpected links/attachments; turn off BT/Wi-Fi when not needed.
  • Weekly: install updates; review password vault; remove unused accounts.
  • Monthly: run HIBP check; ensure browsers saved no passwords.
  • Quarterly: rotate critical passwords; review social privacy; audit startup apps/services.

If You Clicked Something Suspicious (Mini IR Plan)

  • Isolate: disconnect Wi-Fi/cable, enable airplane mode; do not power off if you may need volatile evidence.
  • Do not authenticate: stop entering passwords or MFA approvals; close any prompts.
  • Capture evidence: note time, URL/file, sender, screenshots, and any prompts or downloads.
  • Initial triage: run reputable AV/EDR; check new installs, services, startup items, scheduled tasks, browser extensions.
  • Credentials hygiene: from a clean device, change email, bank, admin, and vault passwords; invalidate active sessions; verify MFA devices.
  • Financial safeguards: if payment data was involved, freeze cards or enable spend limits and alerts; contact bank if needed.
  • Containment & rebuild: for confirmed compromise, back up needed user data, then wipe and reinstall OS; restore from known-good backups only.
  • Reporting: inform IT/Security; for personal cases, consider relevant authorities if fraud occurred; preserve logs.
  • Post-incident: update blockers/filters, patch systems, review user training, and document a timeline and lessons learned.
